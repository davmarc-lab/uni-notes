\subsection{Availability, Consistency, Failure}\label{sec:availability_consistency_failure} % (fold)
Being able to \bf{keep on providing services in spite of failures} is supposed to be one of the main benefit of distributed systems over centralised ones.
\begin{note}[Intuitive Assumption]
    Distributed systems \bf{can be designed} so that if one component of the system fails -- or, it becomes disconnected/partioned -- other components can replace it \bf{to hide failures} from the outside world.
\end{note}
When a system can hide (most of) it failures, it is basically (almost) \it{always working} (highly available). Basic questions in the design of distributed systems: \it{how much failure} can a given system sustain \it{before failure is noticed}?
